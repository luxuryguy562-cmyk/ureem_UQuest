# DESIGN_RULE.md

## 디자인 기준

U-Quest의 현재 화면 기준안은 `mockup.html`이다. 실제 구현은 React/Next 컴포넌트로 분리하되, 사용자 흐름과 정보 배치는 목업을 우선한다.

---

## 모바일 기준

- 기본 검수 폭은 360px 이상 모바일 화면이다.
- 현재 목업 프레임은 390px 폭을 기준으로 한다.
- 텍스트는 먼저 레이아웃으로 보존하고, `ellipsis`는 마지막 안전장치로만 사용한다.
- 하단 네비게이션은 사용자 화면에서만 노출한다.
- 관리자 화면은 운영 밀도가 중요하므로 카드형 목록과 탭 구조를 유지한다.

---

## 범용 UX 기준

- 출석, 캘린더, 요일, 검색, 정렬, 필터, 탭, 토글, 설정 저장 같은 기능은 일반 앱에서 익숙한 패턴을 따른다.
- 범용 기능을 U-Quest만의 방식으로 새로 발명하지 않는다.
- 출석 주간 표시는 `월 화 수 목 금 토 일` 7개 요일을 유지한다.
- 오늘 상태와 출석 완료 상태는 실제 해당 요일 칸에 표시한다.
- 예: 2026년 6월 1일 월요일이면 `월` 칸이 오늘/완료 상태가 된다. 오른쪽 끝에 별도 `오늘` 칸을 만들지 않는다.
- 게임성은 보상 연출, 성장감, 캐릭터/검/나무 표현에서 만든다. 기본 조작 문법은 익숙해야 한다.

---

## 컴포넌트 원칙

- 화면별 컴포넌트는 데이터만 받아 렌더링한다.
- 미션명, 보상명, 금액, 확률, 계수, 정렬순서는 컴포넌트에 직접 쓰지 않는다.
- 이미지/아이콘은 추후 Supabase Storage 또는 `/public/assets` 경로로 교체할 수 있게 데이터 필드로 관리한다.
- 나무 타격, 검 강화, 히든박스는 목업에서는 클라이언트 연출로 유지하되 실제 지급 계산은 서버 결과를 따른다.
