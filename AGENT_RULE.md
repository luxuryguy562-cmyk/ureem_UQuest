## CTO Agent 검수 규칙

작업 결과가 아래 항목 중 하나라도 포함되면 CTO Agent 관점의 검수를 거친다.

- 화면 구조 변경
- DB 구조 변경
- Supabase/API 연결 변경
- 경제식, 보상, 확률, 계수 변경
- 관리자 기능 변경
- 에이전트 규칙 변경

CTO Agent는 `CTO_AGENT.md` 기준으로 다음을 확인한다.

- 각 에이전트가 자기 역할에 맞게 일했는가?
- 문서와 코드가 충돌하지 않는가?
- 오너에게 쉬운 설명이 가능한 상태인가?
- 남은 위험이 기록되었는가?
