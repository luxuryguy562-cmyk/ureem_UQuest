## 현재 사용자 메뉴

PRD 기준 사용자 메뉴:

```text

학습
퀴즈
AX
배지도감
프로필
상점
포인트 이력
```

모바일 하단 탭은 5개로 유지한다.

```text
홈
학습
퀴즈
AX

```

나머지 메뉴는 홈의 메뉴 그리드와 내정보 화면에서 진입한다.

이유:

- 모바일 하단에 8개 탭을 모두 넣으면 터치 영역과 가독성이 무너진다.
- PRD의 메뉴는 모두 제공하되, 사용 빈도에 따라 진입 위치를 나눈다.

## 화면별 목적

### 홈

- 오늘 할 일 확인
- 출석, 학습, 퀴즈, AX 상태 확인
- 현재 포인트, 티어, AX 단계, 캐릭터 단계 확인
- 캐릭터 단계, 퀴즈 티어, AX 로봇을 하나의 성장 쇼케이스 카드로 표시한다.
- 쇼케이스 구도는 상단 퀴즈 티어, 하단 캐릭터와 AX 로봇 배치로 통일한다.
- 캐릭터와 AX 로봇은 같은 카드 안에서 한 세트로 보여주며, 캐릭터가 주 시각 요소가 되고 AX 로봇은 오른쪽 보조 요소로 배치한다.
- 별도 파란 히어로 카드는 사용하지 않고, 프로필형 성장 카드가 홈의 첫 성장 영역이 된다.

### 학습

- 20일 커리큘럼 조회
- 오늘 열리는 Day만 학습 완료 가능
- 학습 완료는 하루 1개 커리큘럼으로 제한
- 학습 완료 후 퀴즈 오픈

### 퀴즈

- Day별 퀴즈 풀이
- 재도전 불가
- 퀴즈 포인트는 정오답과 무관한 제출 보상으로 안내
- 제출 후 정답/해설/오답노트 확인
- 정답률 기반 티어 표시

### AX

- AX/DX 7개 고정 항목 표시
- 사진 올리기 또는 사진 찍기 인증
- 승인 없이 즉시 완료
- 전체 인증 횟수 기반 AX 로봇 성장

### 배지도감

- 출석 배지
- 퀴즈 배지
- 티어 배지
- 희귀 배지
- 획득/미획득/숨김 상태 표시

### 프로필

- 캐릭터 단계
- 티어
- AX 단계
- 배지
- 포인트
- 첫 카드에는 캐릭터, 퀴즈 티어, AX 로봇을 한 칸에 묶어 프로필 쇼케이스로 표시한다.
- 프로필 첫 카드는 상단에 사용자 신분과 퀴즈 티어, 중앙에 큰 캐릭터와 작은 AX 로봇, 하단에 포인트/출석/진행률을 배치한다.
- 최근 획득 배지는 프로필 하단에서 가로 스트립으로 보여주고, 전체 배지도감으로 이동할 수 있게 한다.

### 상점

- 수료 후 오픈
- 포인트로 쿠폰 교환
- 재고, 무제한, 요청, 취소, 발송 상태 표시

### 포인트 이력

- 모든 지급/차감 이력
- 쿠폰 요청 차감
- 쿠폰 취소 원복
- 만료/비활성 소멸 이력

## 권한별 화면

### 신입

온보딩 수행 화면을 사용한다.

### 점장

담당 매장 신입만 조회한다.

가능:

- 신입 목록
- 진행률
- 티어
- AX
- 배지

불가:

- 승인
- 수정
- 포인트 처리
- 쿠폰 처리
- 엑셀 다운로드

### 본사 관리자

전체 운영 화면을 사용한다.

현재 구현 탭:

```text
대시보드
회원
커리큘럼
AX
쿠폰
검증
```

## 현재 코드 구조

```text
src/app/page.tsx
src/components/uquest/uquest-app.tsx
src/lib/mock-data.ts
src/lib/uquest-repository.ts
src/types/uquest.ts
```

`uquest-app.tsx`가 최종 통합 화면의 주 진입점이다.

구버전 화면 단위 컴포넌트는 제거했다.

## UI 검증 기준

- 390px 모바일 폭에서 잘림 없음
- 768px 태블릿 폭에서 오버플로우 없음
- 긴 이름, 긴 쿠폰명, 긴 배지명은 줄바꿈 또는 말줄임 처리
- 하단 탭은 신입 화면에만 표시
- 점장/본사 화면은 운영 밀도를 위해 카드형 목록과 탭 구조 사용
- 가로 스크롤은 원칙적으로 사용하지 않는다.
