# 1. 테스트를 보는 큰 그림

테스트는 아래 질문에 답해야 한다.

```text
신입사원이 오늘 해야 할 온보딩 미션을 이해하는가?
미션 완료 결과로 포인트와 경험치가 명확히 지급되는가?
챕터 진행률이 자연스럽게 올라가는가?
배지/칭호/프레임이 자랑 요소로 작동하는가?
포인트로 쿠폰 교환이 정확히 처리되는가?
```

---

# 2. 기능 흐름 테스트

## 2.1 출석

```text
출석 체크
→ 포인트 지급
→ 경험치 지급
→ 출석 로그 생성
→ 연속 출석 조건 평가
→ 배지 조건 평가
```

확인:

- 같은 날짜에 중복 출석되지 않는다.
- 포인트 원장에 지급 기록이 남는다.
- 경험치 원장에 지급 기록이 남는다.
- 연속 출석 배지가 조건에 맞게 지급된다.

---

## 2.2 미션 완료

```text
미션 완료
→ 자동 승인 또는 관리자 승인 대기
→ 포인트/경험치 확정
→ 챕터 진행률 갱신
→ 컬렉션 획득 조건 평가
```

확인:

- 미션은 하루 제한을 초과해 중복 지급되지 않는다.
- 관리자 승인 미션은 승인 전까지 보상이 확정되지 않는다.
- 완료 후 챕터 진행률이 바뀐다.
- 배지/칭호/프레임 조건이 평가된다.

---

## 2.3 챕터 완료

```text
필수 미션 완료
→ 챕터 완료 조건 충족
→ 챕터 상태 completed
→ 챕터 완료 보상 지급
→ 다음 챕터 활성화
```

확인:

- 필수 미션 조건이 정확하다.
- 선택 미션은 진행률에는 반영되지만 필수 완료 조건과 구분된다.
- 챕터 완료 배지/칭호/프레임이 지급된다.
- 다음 챕터가 자연스럽게 열린다.

---

## 2.4 컬렉션 장착

```text
배지/칭호/프레임 획득
→ 컬렉션 화면 노출
→ 대표 장착
→ 프로필 카드 반영
```

확인:

- 미획득 항목은 장착할 수 없다.
- 장착 변경 이력이 남는다.
- 프로필 카드에 대표 항목이 반영된다.

---

## 2.5 쿠폰 교환

```text
쿠폰 선택
→ 포인트 잔액 확인
→ 재고 확인
→ 포인트 차감
→ 쿠폰 발급
→ 인벤토리 추가
```

확인:

- 포인트가 부족하면 교환되지 않는다.
- 재고가 없으면 교환되지 않는다.
- 포인트 차감과 쿠폰 발급은 동시에 성공하거나 동시에 실패해야 한다.
- 중복 클릭으로 중복 발급되지 않는다.

# 3. 예산 시뮬레이션

기본 계산:

```text
예상 월 지급액 =
활성 직원 수
× 평균 참여율
× 1인 일일 평균 포인트
× 운영일 수
× 포인트 원가
```

예:

```text
24명 × 0.82 × 1,100P × 30일 × 1원 = 649,440원
```

확인:

- 월 기본 예산을 넘지 않는가
- 이벤트 배율 적용 후에도 예산이 안전한가
- 쿠폰 교환율이 예상보다 높을 때 위험도가 표시되는가

---

# 4. RLS 보안 테스트

확인:

- 직원은 본인 데이터만 볼 수 있다.
- 직원은 포인트/경험치/배지/쿠폰을 직접 수정할 수 없다.
- 관리자는 권한 범위 안의 직원만 볼 수 있다.
- 슈퍼관리자만 전체 설정을 수정할 수 있다.
- 퀴즈 정답 정보는 일반 사용자에게 노출되지 않는다.

---

# 5. 오너가 보는 합격 기준

- 홈에서 오늘 할 일이 즉시 보인다.
- 미션 완료 보상이 이해된다.
- 챕터가 온보딩 여정처럼 느껴진다.
- 컬렉션이 직원 간 자랑 요소로 작동한다.
- 포인트와 경험치가 섞이지 않는다.
- 쿠폰 교환이 예산 안에서 안전하게 처리된다.
