# U-Quest 관리자 운영 정책

## 1. 문서 목적

본 문서는 운영자가 수정 가능한 항목과 수정 불가능한 항목을 정의한다.

목적은 다음과 같다.

운영 변경과 개발 변경 구분

대표 요청 대응 기준 마련

시스템 구조 보호

운영 유연성 확보

# 2. 운영 원칙

운영자가 자주 변경할 가능성이 있는 항목은 관리자에서 수정 가능해야 한다.

반대로 시스템 구조 자체는 수정할 수 없어야 한다.

| --- |
| 4주 수료 |
| 20일 커리큘럼 |
| 수료 후 상점 오픈 |
| 포인트 3개월 유효 |
| 1P = 1원 |

# 19. 개발 요청 기준

대표 또는 운영진 요청 발생 시

먼저 아래를 판단한다.

### 운영 수정

관리자에서 해결

예시

배지명 변경

포인트 변경

쿠폰 가격 변경

커리큘럼 수정

### 개발 수정

추가 개발 필요

예시

배지 구조 변경

권한 구조 변경

수료 정책 변경

티어 구조 변경

캐릭터 성장 단계 변경

운영으로 해결 가능한 요구사항은 개발 요청으로 처리하지 않는다.
